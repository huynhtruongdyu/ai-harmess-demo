# Write User Guide Prompt

## Context
Write a user guide for {feature_name} that helps end users understand and use the feature effectively.

## Feature Details
- **Feature**: {feature_name}
- **Release Version**: {version}
- **Target Users**: {target_audience}
- **Key Functionality**: {key_functionality}
- **Prerequisites**: {prerequisites}

## Guide Structure

### 1. Overview
- What does this feature do?
- Why would a user use it?
- What problem does it solve?

### 2. Getting Started
- Prerequisites (accounts, permissions, configurations)
- Step-by-step setup instructions
- Screenshots of key screens

### 3. How-To Guides
**Topic 1: {topic_1}**
1. {step_1}
2. {step_2}
3. {step_3}

**Topic 2: {topic_2}**
1. {step_1}
2. {step_2}

### 4. Best Practices
- {best_practice_1}
- {best_practice_2}

### 5. Troubleshooting
| Issue | Likely Cause | Solution |
|-------|-------------|----------|
| {issue} | {cause} | {solution} |

### 6. FAQs
- **Q**: {question}
  **A**: {answer}

## Writing Guidelines
- Write for the target audience's technical level
- Use numbered steps for procedures
- Include annotated screenshots for UI features
- Define jargon in plain language
- Keep paragraphs short (3-4 sentences max)
- Use bullet points for lists, numbered steps for procedures
- Include a "What's Next" section linking to related guides
