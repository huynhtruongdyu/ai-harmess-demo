# New Feature Implementation Checklist

## Pre-Development
- [ ] Story understood and acceptance criteria reviewed
- [ ] Architecture/design reviewed (if complex)
- [ ] Test plan outlined
- [ ] Dependencies identified

## Development
- [ ] Create feature branch from main
- [ ] Implement domain/entity changes
- [ ] Implement service/business logic
- [ ] Implement API endpoints (if applicable)
- [ ] Implement UI components (if applicable)
- [ ] Add input validation
- [ ] Add error handling
- [ ] Add logging
- [ ] Write unit tests (80%+ coverage)
- [ ] Write integration tests
- [ ] Run linter and type checker
- [ ] Run full test suite
- [ ] Update API documentation (if applicable)
- [ ] Create PR with description linking to story

## Review
- [ ] Code review by peer
- [ ] All review comments addressed
- [ ] CI pipeline passes
- [ ] QA verification (if applicable)
- [ ] Product demo (if applicable)

## Post-Merge
- [ ] Deploy to staging
- [ ] Verify in staging environment
- [ ] Update documentation
- [ ] Update changelog
- [ ] Monitor deployment for 24 hours
